MCP · Retrieval · Docker

BetterAI is a rule-retrieval server for coding agents.

One MCP tool — retrieve_context — takes a task description and returns the handful of rules, skills, and memories that matter, scored by a grep + MiniLM-embedding hybrid. This map shows every component and traces a single request from the wire to the audit log.

At a glance

The seven components

  • 01

    HTTP + bearer transport

    Streamable HTTP MCP transport. Every request carries a bearer token, checked before any work happens.

  • 02

    Connection limiter

    Bounds in-flight requests so a burst degrades gracefully instead of exhausting the box.

  • 03

    LRU cache

    Keyed on task + repo scope. A hit returns ranked context without touching disk or the model.

  • 04

    Retrieval orchestrator

    The core. Runs grep over the corpus for lexical hits, MiniLM embeddings for semantic ones, then merges, ranks, and returns top-k.

  • 05

    Repo-scope detection

    Infers the repo’s domain so a frontend task never gets backend-only rules.

  • 06

    Markdown corpus reader

    Parses rule, skill, and memory markdown into scorable, retrievable documents.

  • 07

    JSONL audit log

    Append-only record of every retrieval: query, scope, matches, scores — grep-friendly and replayable.

  • 08

    CLI

    Drives the orchestrator from a terminal for local debugging and corpus authoring — same code path, no MCP client needed.
