About

Systems that stay calm under load — and under scrutiny.

I started in distributed systems — queues, caches, and the kind of connection-limiting that decides whether a Tuesday is quiet or a pager storm. When LLM tooling arrived, I found the same problems wearing new clothes: retrieval is search, an MCP server is just a daemon with a bearer token, and an agent fleet is a thread pool that occasionally argues with you.

My bias is toward boring, legible infrastructure: explicit transports, audit logs you can grep, scoring you can explain in one sentence, and caches that admit when they’re cold. I’d rather ship a 38ms hybrid retriever I can reason about than a magic black box that wins a benchmark and loses on-call.

Selected work

Three things I built and would build again.

  • 01

    BetterAI

    MCP rule-retrieval server · Lead engineer

    A Dockerized MCP server that hands coding agents the right rules at the right moment. Grep + MiniLM-embedding hybrid scoring over a markdown corpus, an LRU cache and connection limiter in front, a JSONL audit log behind, and repo-scope detection so a frontend repo never gets backend rules. p95 sits at 38ms warm.

    HTTP+bearer MCP · TypeScript · MiniLM · Docker

  • 02

    Driftwatch

    Embedding-drift monitor · Creator

    A sidecar that watches production embedding distributions and fires when the index quietly goes stale — the failure mode nobody alerts on. It re-embeds a rolling sample, tracks cosine drift against a frozen baseline, and opens a PR with the offending documents attached so the fix is one click, not one investigation.

    Python · pgvector · GitHub Actions

  • 03

    Lanternd

    Agent fleet scheduler · Creator

    A scheduler for running dozens of coding agents in parallel without them trampling each other. Per-repo worktree leases, a token-budget governor, and a connection limiter that backs off cleanly instead of thrashing. It turned “spawn 40 agents and pray” into a load I can actually reason about on a graph.

    Rust · tokio · SQLite · Prometheus
