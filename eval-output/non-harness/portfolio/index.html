About

I'm an engineer focused on the unglamorous half of AI: the retrieval orchestration, the connection limiters, the cache eviction policy, and the JSONL audit trail that lets you answer "why did the model see that?" three weeks later.

My recent work centers on the Model Context Protocol — building HTTP transports with bearer auth, hybrid scorers that combine grep precision with MiniLM embedding recall, and markdown corpus readers that turn a folder of rules and memories into a queryable knowledge surface for coding agents.

I care about systems that degrade gracefully, observability you don't have to bolt on later, and developer ergonomics — a good CLI is a feature, not an afterthought.

Selected projects

BetterAI — MCP rule-retrieval server

A Dockerized MCP server that serves rules, skills, and memories to coding agents over an HTTP + bearer transport. A retrieval orchestrator blends grep matching with MiniLM-embedding similarity for hybrid scoring, fronted by an LRU cache and connection limiter, with a JSONL audit log and repo-scope detection so agents only see rules relevant to the repo they're in.

  • TypeScript
  • MCP
  • MiniLM
  • Docker

Hybrid retrieval scorer

A standalone scoring library that reconciles two retrieval signals that usually fight each other: exact lexical hits from grep and fuzzy semantic neighbors from embeddings. It normalizes both into a single ranked list, with tunable weighting and a tie-break that favors precision on short queries.

  • Embeddings
  • BM25-style scoring
  • Benchmarked

Audit-first request pipeline

A request middleware layer that writes a structured JSONL audit record for every retrieval — query, matched documents, scores, cache hit/miss, and latency — so you can replay exactly what an agent was given. Pairs with a connection limiter that sheds load before the embedding model becomes the bottleneck.

  • JSONL
  • Backpressure
  • Observability
