System Architecture Map

BetterAI

A Dockerized MCP rule-retrieval server. It exposes a single retrieve_context tool over an HTTP + bearer MCP transport, then ranks rules, skills, and memories from a markdown corpus using hybrid grep + MiniLM-embedding scoring — fronted by an LRU cache and connection limiter, recorded in a JSONL audit log, and scoped to the calling repository.

Component map

How a request travels: an agent calls the MCP tool, the request is authenticated and limited, the orchestrator checks cache then scores the corpus, and the result is logged before it returns.

The pieces at a glance

HTTP + Bearer transport

MCP over HTTP with bearer-token auth. The single front door for every request.

Retrieval orchestrator

Coordinates cache, scope, scoring, and audit to answer retrieve_context.

Hybrid scorer

Combines grep lexical hits with MiniLM embedding similarity into one ranked list.

Markdown corpus reader

Loads rules, skills, and memories from markdown into a queryable document set.

LRU cache + limiter

Caches query results and sheds load before the embedding model saturates.

JSONL audit log

Append-only record of every query, its matches, scores, and latency.

Repo-scope detection

Restricts results to rules relevant to the calling repository.

CLI

Local entrypoint to run the server and query the corpus without an agent.
