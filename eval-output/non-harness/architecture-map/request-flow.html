Walkthrough

A retrieve_context request, step by step

Follow one query from the agent's call all the way to the ranked response.

  1. 1

    Agent calls the tool

    A coding agent (or the CLI) invokes retrieve_context with a query string over the HTTP MCP transport, carrying its bearer token.

  2. 2

    Authenticate & admit

    The transport verifies the bearer token. The connection limiter checks in-flight capacity and admits the request, or sheds/queues it if the system is saturated.

  3. 3

    Resolve repo scope

    The orchestrator detects which repository the request belongs to so retrieval can be narrowed to relevant rules and memories.

  4. 4

    Check the LRU cache

    The orchestrator looks up the normalized query + scope. On a hit, it returns the cached ranked result immediately and skips to step 7.

  5. 5

    Read corpus & score (cache miss)

    The markdown corpus reader supplies candidate documents. The hybrid scorer runs a grep lexical pass and a MiniLM embedding pass, then merges them into one normalized ranking.

  6. 6

    Cache the result

    The freshly ranked result is written back into the LRU cache (evicting the oldest entry if full) so identical follow-up queries are cheap.

  7. 7

    Write the audit record

    The orchestrator appends a JSONL line: the query, matched documents, scores, cache hit/miss, and latency — a complete, replayable record.

  8. 8

    Respond to the agent

    The ranked rules, skills, and memories travel back out through the transport to the agent as the tool result.

Two paths, one shape

The cache hit path (steps 1-4, 7-8) and the cache miss path (all 8 steps) differ only in whether scoring runs. Both are authenticated, scoped, and audited — nothing returns to the agent without a log line.
