cli — 7 verbs for authoring + dogfooding

Tier: Operator · Files: src/cli/{main,init,validate,status,why,replay,gate,new}.ts + src/cli/_shared/{frontmatter,repo-root}.ts + bin/betterai · Last commit: f4f8801 (Wave 3)

7 verbs designed to make BetterAI usable without leaving the terminal. Read-only verbs (validate, status --offline, why --offline) work without the container running — they read ~/.betterai/ directly from disk. Per the BetterAI repo rule cli-read-ops-work-offline.

The 7 verbs

VerbPurposeNeeds container?Source
betterai init [--seed]Scaffold ~/.betterai/; copy seed corpus from /opt/betterai/seed-corpus; drop welcome taskyes (run inside container)init.ts
betterai validateSchema-validate global AND repo corporaNO (offline)validate.ts
betterai status [--offline]Show counts + recent activity; --offline reads disk onlypartial (offline mode = no)status.ts
betterai why <task> [--context <path>]Simulate retrieve_context against a hand-given context; explain routingyes (uses MCP)why.ts
betterai replay --since 7dWeekly audit-log digest: fires + missed + behavior changesNO (reads audit JSONL)replay.ts
betterai gate --week NSelf-verify Phase 1.0 dogfooding criteria (≥5 fires × ≥3 behavior changes × 5 days)NO (audit JSONL)gate.ts
betterai new {rule,skill,memory} [--scope repo|global]Scaffold a new file with frontmatter pre-stubbedNO (writes file)new.ts
