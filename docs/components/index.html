How docs stay current

The harness keeps docs in sync via .githooks/pre-commit, which runs scripts/update-component-docs.mjs. The script:

  • reads git diff --cached --name-only for staged files
  • maps each path to its component (per the mapping in scripts/update-component-docs.mjs)
  • updates the “Recent changes” table at the top of Current Status for each touched component
  • re-stages the modified doc files so they ship with the commit

Install with git config core.hooksPath .githooks after cloning. The CI step npm run docs:check verifies docs/component HTMLs reflect the last commit.
