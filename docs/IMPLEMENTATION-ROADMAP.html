BetterAI v4 — Implementation Roadmap

A Dockerized rule corpus + MCP retrieval service that injects relevant code-quality/design/maintainability rules into any AI agent’s context before the agent writes or plans code. VSCode is the human surface. Personal toolkit, not a product. The corpus is the moat.

This file consolidates the v4 design (APPROVED) and the three reviews from 2026-06-09 (corpus migration eng review, multi-agent + skills/memories eng review, DX review) into a single implementation map. Source files referenced in §References.

Phase 1.0 — Container + MCP Wedge (macOS-only)

2–3 weeks solo
  • TS scaffold + Dockerfile (single-arch linux/arm64 only)
  • docker-compose.yml with bearer-token + UID/GID + 5 volume mounts
  • MCP tools (HTTP/SSE transport only — stdio dropped per multi-agent eng review CRIT-1): retrieve_rules, retrieve_skills, retrieve_memories, retrieve_context, record_memory, explain_rule, check_file
  • Grep-only retrieval (no embeddings); regex-only check_file (no ast-grep, no shell, no ts-module)
  • Audit JSONL with extended schema: parent_agent_session_id, subagent_class, retrieved_kinds (multi-agent eng review §1.4)
  • context_hash LRU cache (256 entries, 60s TTL) + connection limiter (max 16 in-flight)
  • Auto-retrieve lever (a): install script writes CLAUDE.md preamble OR sets BETTERAI_SYSTEM_PROMPT_PREFIX
  • betterai-cli v1: init / validate / status / why / replay (DX review DX-FIX-5)
  • Welcome task (DX review DX-FIX-1): ~/.betterai/welcome-task/README.md with a deliberately rule-triggering prompt
  • Install script forward from Phase 1.5 (DX review DX-FIX-2): docker-compose + MCP config + bearer token + welcome-task in one command

DOGFOODING GATE (Week 2) 5 consecutive engineering days using Claude Code + BetterAI on real work. ≥5 rules fire AND ≥3 visible behavior changes. Else re-author rules — schema isn’t the problem, content is.

Phase 1.5 — Polish + Cross-Platform

3–4 weeks
  • Embedding index (MiniLM via @xenova/transformers, sqlite-vec); activate when any single domain exceeds 30 rules
  • check.kind=ast-grep support
  • Multi-arch image (amd64 + arm64) via buildx + GitHub Actions matrix runners
  • GitHub Releases CI/CD pipeline
  • macOS launchd / Linux systemd user-unit auto-start
  • Auto-retrieve lever (b): server emits missed_retrieval audit events
  • Auto-retrieve lever (c): optional ~/.claude/skills/betterai-auto-retrieve/SKILL.md
  • Three-tier error taxonomy [BAI-NNN] with Stripe-shaped MCP error responses (DX review DX-FIX-9)
  • Docs: README.md + docs/AUTHORING.md + docs/DEBUGGING.md (DX review DX-FIX-12)
